# 部署检查清单

在将系统部署到生产环境之前，请逐项检查以下内容：

## 📋 部署前检查

### 1. 环境准备
- [ ] 服务器已准备就绪（CPU 2核+，内存 4GB+，磁盘 20GB+）
- [ ] 已安装Docker和Docker Compose（如使用Docker部署）
- [ ] 已安装Python 3.9+（如直接部署）
- [ ] 已安装MySQL 8.0+
- [ ] 已配置域名和DNS解析（如需外网访问）

### 2. 数据库配置
- [ ] 已创建MySQL数据库
- [ ] 已创建数据库用户并授权
- [ ] 数据库密码使用强密码（至少16位）
- [ ] 数据库只允许应用服务器IP访问
- [ ] 已测试数据库连接

### 3. 安全配置
- [ ] 已生成随机的SECRET_KEY（至少32位）
- [ ] 已修改.env文件中的所有默认密码
- [ ] 已删除所有测试文件（test_*.py, verify_*.py等）
- [ ] 已配置.gitignore，不提交敏感信息
- [ ] 已配置防火墙规则
- [ ] 已配置SSL证书（生产环境必须）

### 4. 应用配置
- [ ] 已复制.env.example为.env
- [ ] 已正确配置.env文件中的所有变量
- [ ] 已设置APP_ENV=production
- [ ] 已设置APP_DEBUG=false
- [ ] 已配置CORS_ORIGINS（如需跨域）

### 5. 文件检查
- [ ] 已检查所有必需文件存在
  - [ ] main.py
  - [ ] requirements.txt
  - [ ] Dockerfile
  - [ ] docker-compose.yml
  - [ ] .env
  - [ ] static/目录及所有HTML文件
- [ ] 已删除不需要的文件
  - [ ] test_password.py
  - [ ] verify_password.py
  - [ ] test_bcrypt.py
  - [ ] migrate_db.py
  - [ ] migrate_custom_forms.py

## 🚀 部署过程检查

### 6. 部署执行
- [ ] 已选择合适的部署方式
  - [ ] Docker Compose（推荐）
  - [ ] Docker
  - [ ] 直接部署
- [ ] 已成功构建镜像或安装依赖
- [ ] 已成功启动服务
- [ ] 服务运行正常，无错误日志

### 7. 网络配置
- [ ] 应用端口已正确映射（默认8000）
- [ ] 已配置Nginx反向代理（推荐）
- [ ] 已配置HTTPS和SSL证书
- [ ] 已配置WebSocket支持
- [ ] 已测试HTTP/HTTPS访问

### 8. 功能测试
- [ ] 可以正常访问登录页面
- [ ] 可以使用默认管理员账号登录
- [ ] 可以创建新用户
- [ ] 可以创建工单
- [ ] 可以接收实时通知
- [ ] WebSocket连接正常
- [ ] 所有页面功能正常
