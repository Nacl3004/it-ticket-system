# 工单分类与IT人员配置说明

## 功能概述

系统支持将不同的工单分类自动分配给特定的IT人员处理,实现工单的智能分配和流程管理。

## 核心功能

### 1. 工单分类与IT人员映射

管理员可以配置每个工单分类对应的IT处理人员:
- 支持一个分类对应多个IT人员（按优先级排序）
- 支持一个IT人员负责多个分类
- 支持动态启用/禁用配置

### 2. 自动分配机制

当用户提交工单时:
1. 系统根据工单分类查找对应的IT人员配置
2. 按优先级选择最高优先级的IT人员
3. 自动将工单分配给该IT人员
4. 发送实时通知给被分配的IT人员
5. 如果没有配置对应的IT人员,则通知所有IT人员

### 3. 实时消息推送

工单处理过程中的实时通知:
- **工单创建时**: 通知对应的IT人员或所有IT人员
- **工单认领时**: 通知工单提交人
- **工单处理中**: IT人员可以添加备注,实时推送给提交人
- **工单完成时**: 通知提交人确认
- **工单结单时**: 通知IT人员

## 数据库表结构

### category_it_mapping 表

工单分类与IT人员映射表:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键ID |
| category_id | INT | 工单分类ID |
| it_user_id | INT | IT人员用户ID |
| priority | INT | 优先级（数字越大优先级越高） |
| is_active | TINYINT | 是否启用（1=启用，0=禁用） |
| created_at | TIMESTAMP | 创建时间 |

### ticket_workflow_nodes 表

工单流程节点配置表（预留扩展）:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键ID |
| category_id | INT | 工单分类ID |
| node_name | VARCHAR(100) | 节点名称 |
| node_order | INT | 节点顺序 |
| node_type | ENUM | 节点类型（auto_assign/manual_claim/process/complete/close） |
| required_role | VARCHAR(50) | 需要的角色类型 |
| is_active | TINYINT | 是否启用 |
| created_at | TIMESTAMP | 创建时间 |

## API接口说明

### 1. 获取配置列表

**接口**: `GET /api/category-it-mapping`

**权限**: 需要登录

**返回示例**:
```json
{
  "mappings": [
    {
      "id": 1,
      "category_id": 1,
      "category_name": "硬件故障",
      "it_user_id": 5,
      "it_user_name": "张三",
      "it_username": "zhangsan",
      "priority": 10,
      "is_active": 1,
      "created_at": "2024-01-01 10:00:00"
    }
  ]
}
```

### 2. 创建配置

**接口**: `POST /api/category-it-mapping`

**权限**: 超级管理员、管理员

**参数**:
- `category_id`: 工单分类ID（必填）
- `it_user_id`: IT人员用户ID（必填）
- `priority`: 优先级（可选，默认0）

**请求示例**:
```json
{
  "category_id": 1,
  "it_user_id": 5,
  "priority": 10
}
```

### 3. 删除配置

**接口**: `DELETE /api/category-it-mapping/{mapping_id}`

**权限**: 超级管理员、管理员

**说明**: 软删除，将is_active设置为0

## 使用场景示例

### 场景1: 硬件故障专人负责

**配置**:
- 工单分类: 硬件故障
- IT人员: 张三（硬件维修专家）
- 优先级: 10

**效果**: 所有硬件故障工单自动分配给张三

### 场景2: 多人负责同一分类

**配置**:
- 工单分类: 软件故障
- IT人员1: 李四（优先级: 10）
- IT人员2: 王五（优先级: 5）

**效果**: 软件故障工单优先分配给李四,如果李四不可用,可以手动调整配置

### 场景3: 一人负责多个分类

**配置**:
- 张三负责: 硬件故障、打印机故障、网络故障

**效果**: 这三类工单都会自动分配给张三

## 工单处理流程

### 标准流程

1. **用户提交工单**
   - 填写工单信息（标题、分类、描述等）
   - 系统生成工单编号
   - 根据分类自动分配IT人员（如有配置）

2. **IT人员接收通知**
   - WebSocket实时推送
   - 浏览器桌面通知
   - 系统内通知列表

3. **IT人员处理工单**
   - 查看工单详情
   - 开始处理（可添加备注）
   - 实时通知提交人处理进度

4. **IT人员完成工单**
   - 填写解决方案
   - 标记为已完成
   - 通知提交人确认

5. **用户确认结单**
   - 查看解决方案
   - 确认问题已解决
   - 结束工单流程

### 实时消息推送时机

| 操作 | 推送对象 | 消息类型 | 消息内容 |
|------|----------|----------|----------|
| 创建工单 | 对应IT人员/所有IT | new_ticket_assigned / new_ticket | 新工单已分配/新工单提醒 |
| 认领工单 | 工单提交人 | ticket_claimed | 工单已被认领 |
| 开始处理 | 工单提交人 | ticket_processing | 工单处理中（含备注） |
| 完成工单 | 工单提交人 | ticket_completed | 工单已完成 |
| 结单 | IT人员 | ticket_closed | 工单已结单 |

## 配置建议

### 1. 按专业领域分配

根据IT人员的专业技能配置对应的工单分类:
- 硬件专家 → 硬件故障
- 网络工程师 → 网络故障
- 软件开发 → 软件故障

### 2. 设置合理的优先级

- 主要负责人: 优先级 10
- 备用负责人: 优先级 5
- 临时支援: 优先级 1

### 3. 定期审查配置

- 根据工单处理情况调整配置
- 平衡各IT人员的工作负载
- 及时更新人员变动

## 注意事项

1. **权限要求**: 只有超级管理员和管理员可以配置映射关系
2. **优先级规则**: 数字越大优先级越高,系统选择最高优先级的IT人员
3. **软删除机制**: 删除配置不会真正删除数据,只是将is_active设置为0
4. **实时通知**: 需要浏览器支持WebSocket和桌面通知权限
5. **备注功能**: IT人员在处理工单时可以添加备注,实时推送给用户
